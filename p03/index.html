What is a Graph?

For the purpose of this guide, a graph is not what you likely have in mind as a graph. As opposed to the typical idea of bar graphs or pie charts, we are talking about a simply a collection of points connected to one another. The points, which can contain information, are known as nodes, and the connections between them are known as edges. The field of mathematics known as graph theory studies these graphs and algorithms and theorems that are helpful in traversing the data contained within them.

What are graph databases used for?

Graph data bases are used for many different situations.For example, Walmart used a graph database to implement their reccomendation engine based on the relationships with past searches. Also, the US Army uses a graph database to track data about their machinery so they know when they need to replace parts. Graph databases are also very useful in fraud detection.

Pros and Cons

Pros
  • Flexibility in how you define your nodes and relationships
  • Fast relationship-based search and indexing
  • Graph databases have specific functionalities to store/manage graphs
Cons
  • Not always efficient based on the relationships between data pieces
  • All data stored on one server so not very scalable
  • Can't perform mass analytics

Graph Algorithms

The mathematical field of graph theory has produced a number of algorithms that are immensely helpful when attempting to quickly traverse a graph database. The most famous of these is Dijkstra's Algorithm Which can find the shortest path connecting two nodes in a graph, using some sort of contained data about the "length" of each edge. In a similar vein, the Bellman-Ford Algorithm finds and stores the shortest path from one node in a graph to any other node connected to it.

While the mathematics behind these algorithms is extremely interesting, we won't explain that here. We choose not to because of the fact that in order to use a graph database, you don't need to know how these algorithms work. These algorithms are built into graph database systems, under names like 'shortest-path', so that you can use the full potential of the graph database without being restrained by the details of algorithms.

neo4j

neo4j is one of the most well established and widely used services for building a graph database. It is free to use, has a great number of tutorials on their website to help understand the basics of graph theory and get into building your own graph database. There are paid versions available for larger scale operations by large companies, but because there is a free version available, you can play around with the concepts of graph theory on your own.

neo4j uses a language of their own creation, known as Cypher, to allow you to interact with your graph database. Without getting too much into the details, its syntax is based loosely on ASCII art representing the edges with dashes and arrows and nodes with brackets. Graph algorithms are built into it with simple command syntaxes. Rather than going in depth with a tutorial here, we recommend you check out the tutorials provided by neo4j, which both help develop an understanding of their own language, as well as develop a further understanding of the concepts behind graph databases.
